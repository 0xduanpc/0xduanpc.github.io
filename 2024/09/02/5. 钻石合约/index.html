5. 钻石合约

Posted on 2024-09-02

1. Diamond Pattern(EIP-2535)

1.1 解决的问题

  • 单一地址支持无限合约功能。使用单一地址来处理合约功能，使得合约的部署、测试以及与其他智能合约、软件和用户界面的集成变得更加便捷。
  • 合约超过 24KB 的最大大小限制。如果你的合约包含多个相关功能，将它们集中在一个单一合约或地址中可能更为合理。钻石合约没有最大合约大小限制。
  • 钻石合约提供组织合约代码和数据的方式。如果你需要构建一个功能丰富的合约系统，钻石合约提供了一种系统化的方法，可以将不同功能模块化，并在需要时高效地共享数据。
  • 钻石合约支持功能升级。可升级的钻石合约可以通过添加、替换或移除功能来进行升级。由于钻石合约没有最大合约大小限制，因此可以不断增加功能。钻石合约可以是可升级的，也可以是不可升级的，并且可以在以后移除其升级功能。

1.2 三个基础方法

  1. 回退函数（Fallbacks）
  2. 委托调用（Delegate Calls）
  3. 存储布局（Storage / Layout）

2. Fallbacks

在调用合约中不存在的函数时，会调用 fallback 方法兜底

3. DelegateCalls

通过 address 和方法名调用外部合约，注意 uint 要改写为 uint256

4. Storage / Layout

  • 公有变量存在钻石合约，下游合约实现具体方法
  • 以太坊网络会尽可能的把变量紧密的排布起来
  • 可以通过构造 struct 合约解决变量编排问题
  • 但是会覆盖被继承合约的变量，所以需要指定 Storage 结构体的 slot 来解决
  • 但 struct 内部的属性顺序不能变，否则会出现错误

5. Diamond Contract 实践

  • EIP2535 标准案例：diamond-3-hardhat
  • DiamondCutFacet 用于添加、移除、更换钻石合约使用的 functions
  • 用 diamondCutFacet 的引用初始化钻石合约，钻石合约去 diamondCutFacet 里调用使用的方法
